This page assumes you've already [installed the dependency](/installation). Every compressor lives on a single `Kompressor` instance.

## Create a Kompressor

```kotlin
import co.crackn.kompressor.createKompressor

val kompressor = createKompressor()
```

<Note>
  Each compressor (`image`, `video`, `audio`) is initialised lazily ‚Äî only the ones you access are created. Safe to keep a single `Kompressor` around for the lifetime of the app.
</Note>

## Compress an image

```kotlin
import co.crackn.kompressor.io.MediaDestination
import co.crackn.kompressor.io.MediaSource

val result = kompressor.image.compress(
    input  = MediaSource.Local.FilePath("/path/to/photo.jpg"),
    output = MediaDestination.Local.FilePath("/path/to/photo_compressed.jpg"),
)

result.onSuccess { r ->
    println("Compressed: ${r.outputSize / 1024} KB in ${r.durationMs}ms")
}.onFailure { error ->
    println("Error: ${error.message}")
}
```

<Note>
  `MediaSource` and `MediaDestination` are sealed hierarchies ‚Äî you can also pass a `Uri` (Android), `NSURL` / `PHAsset` (iOS), an `okio.Source` / `okio.Sink`, or raw `ByteArray`. See [the I/O model](/concepts/io-model) for the full set of builders.
</Note>

## Compress a video

```kotlin
val result = kompressor.video.compress(
    input  = MediaSource.Local.FilePath("/path/to/video.mp4"),
    output = MediaDestination.Local.FilePath("/path/to/video_compressed.mp4"),
    onProgress = { progress -> updateProgressBar(progress.fraction) },
)
```

The `onProgress` callback receives a `CompressionProgress` with `phase` and `fraction` (`0.0`‚Äì`1.0`). Use `progress.phase` to swap UI labels between materializing, compressing, and finalizing ‚Äî see [the I/O model](/concepts/io-model#progression-phases).

## Compress audio

```kotlin
val result = kompressor.audio.compress(
    input  = MediaSource.Local.FilePath("/path/to/recording.wav"),
    output = MediaDestination.Local.FilePath("/path/to/recording.m4a"),
)
```

Audio output is always **AAC in an `.m4a` container**. Inputs can be WAV, MP3, M4A / AAC, FLAC, OGG / Opus, AMR, or the audio track of an MP4 ‚Äî anything the platform's default extractors can open.

## Check the result

Every `compress(...)` call returns `Result<CompressionResult>`:

```kotlin
data class CompressionResult(
    val inputSize: Long,        // bytes
    val outputSize: Long,       // bytes
    val durationMs: Long,       // milliseconds
) {
    val compressionRatio: Float        // outputSize / inputSize
    val isSmallerThanOriginal: Boolean // outputSize < inputSize
}
```

<Warning>
  Re-encoding an already-compressed file can produce a **larger** output ‚Äî especially JPEG at a higher quality than the source, or AAC at a higher bitrate than the source. Check `isSmallerThanOriginal` and discard the output when compression would hurt instead of help.
</Warning>
