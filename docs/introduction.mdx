Kompressor is a Kotlin Multiplatform library that compresses **images**, **videos**, and **audio** on Android and iOS through the platform's own hardware-accelerated media stack. There is no bundled codec binary, no FFmpeg, no CPU-only fallback â€” just a thin Kotlin layer over `BitmapFactory`, Media3 `Transformer`, `AVAssetExportSession`, and friends.

```kotlin
val result = kompressor.video.compress(
    inputPath  = "/path/to/video.mp4",
    outputPath = "/path/to/out.mp4",
    onProgress = { fraction -> updateProgressBar(fraction) },
)
```

## Why Kompressor

<CardGroup cols={2}>
  <Card title="Zero binary overhead" icon="feather">
    Nothing is shipped alongside your APK / IPA. The library delegates to the decoders and encoders the OS already provides.
  </Card>
  <Card title="Hardware-accelerated" icon="bolt">
    Transcodes run on the same hardware paths platform media apps use â€” MediaCodec on Android, VideoToolbox / AudioToolbox on iOS.
  </Card>
  <Card title="One API, two platforms" icon="code">
    A single `Kompressor` interface in `commonMain` with `expect`/`actual` wiring. Write compression logic once.
  </Card>
  <Card title="Structured concurrency" icon="arrows-rotate">
    Compression is a `suspend` function. Cancel the coroutine to abort the transcode; progress arrives through a suspend callback.
  </Card>
</CardGroup>
