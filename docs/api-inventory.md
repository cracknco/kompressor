# Public API Inventory

## Source of truth

The committed ABI baseline — `kompressor/api/kompressor.api` — is the authoritative
inventory of every public JVM signature Kompressor ships. It is generated by
[Kotlin Binary Compatibility Validator](https://github.com/Kotlin/binary-compatibility-validator)
via `./gradlew apiDump` and verified on every PR by `./gradlew apiCheck` (see
[`docs/api-stability.md`](api-stability.md) for the drift gate + S/E/I review checklist).

Experimental opt-in markers live in `.kotlin_metadata` (not in the JVM signature), so the
authoritative list of experimental symbols is in the source code itself — grep for
`@ExperimentalKompressorApi` under `kompressor/src/commonMain/`.

## Stability tiers

**Stable (S)** — covered by SemVer 2.0.0 from 1.0.0 onward. Source- and binary-breaking
changes require a MAJOR release. Every stable symbol carries KDoc enforced by Detekt's
`UndocumentedPublicClass/Function/Property` rules.

**Experimental (E)** — annotated `@ExperimentalKompressorApi`. Opt-in required at the call
site. May change or be removed in any MINOR release. Consumers see a WARNING from the
compiler until they `@OptIn(ExperimentalKompressorApi::class)`.

**Internal (I)** — `internal` visibility. Invisible to consumers, no stability guarantee,
absent from the ABI dump by construction.

## Current `@ExperimentalKompressorApi` surface

Pre-1.0 incubating symbols (grep-verifiable from `kompressor/src/commonMain/`):

- `AudioChannels.FIVE_POINT_ONE`, `AudioChannels.SEVEN_POINT_ONE` — per-device AAC encoder
  coverage is still being validated; surround requests that the platform encoder rejects
  surface as a typed [`AudioCompressionError`] at compress time.
- `AudioCompressionConfig.audioTrackIndex` — multi-track selection semantics (stream-copy
  restrictions on Android, probe ordering across containers) are still being stabilised.
- `SourceMediaInfo.audioTrackCount` — paired with `audioTrackIndex`; out together.
- `ImageFormat.HEIC`, `ImageFormat.AVIF` — decoding works on Android API 30+ (HEIC) / 31+
  (AVIF) and iOS 15+ / 16+; encoding is platform-gated and not yet universally available.
- `DynamicRange.HDR10`, `VideoPresets.HDR10_1080P` — HEVC Main10 encoder coverage varies
  per device; the typed-error contract is stable but support detection isn't.

## Module-level opt-in for test code

`kompressor/build.gradle.kts` opts every test source set into
`ExperimentalKompressorApi` via `languageSettings.optIn(...)`, so individual test classes
never need per-class `@OptIn`. Production source sets (`commonMain`, `androidMain`,
`iosMain`) must opt in explicitly — an accidental use of experimental API from stable
code will fail compilation with a WARNING-level opt-in requirement.

## Reviewing ABI diffs

The S/E/I review checklist (Source-compatible? Experimental? Intentional?) lives in
[`docs/api-stability.md`](api-stability.md). When a PR changes
`kompressor/api/kompressor.api`, reviewers walk that checklist before approving.
